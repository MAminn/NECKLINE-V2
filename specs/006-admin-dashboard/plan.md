# Implementation Plan: Admin Dashboard

**Branch**: `006-admin-dashboard` | **Date**: 2026-06-05 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/006-admin-dashboard/spec.md`

---

## Summary

Build a full 10-tab protected admin portal that matches the recovered `NicklineAdminPortal.tsx` UI design. The admin frontend lives at `/admin` inside the existing Next.js app, gated by role-based auth. The backend adds new Mongoose models (`Testimonial`, `HeaderSlide`, `HowToApply`), a `fulfillmentStatus` field on `Order`, and a suite of admin API endpoints. All admin mutations are audit-logged. Public content endpoints (`/testimonials`, `/header-slides`, `/how-to-apply`) serve the storefront without authentication.

---

## Technical Context

**Language/Version**: TypeScript (Next.js frontend), JavaScript (Express API), Node 20

**Primary Dependencies**: Next.js 15, React 18, Tailwind CSS, Framer Motion, Mongoose 8, Express 4, Lucide React (icons — already installed)

**Storage**: MongoDB (existing) — 3 new collections: `testimonials`, `headerslides`, `howtoapplies`

**Testing**: Jest (existing API test suites), manual E2E via quickstart.md

**Target Platform**: Web — desktop-first admin UI; same deployment as storefront

**Performance Goals**: Dashboard KPI cards load in < 2s; paginated lists at 8–20 rows; no unbounded scans

**Constraints**: Must match recovered UI design token-for-token. No additional npm packages unless essential.

**Scale/Scope**: MVP scale (~200 products, ~500 orders, ~150 customers); MongoDB aggregation is acceptable without a dedicated analytics store.

## Implementation Phases

### Phase A — Backend Models & Content Endpoints
New models + public read + admin CRUD for Testimonial, HeaderSlide, HowToApply.

### Phase B — Admin Infrastructure Endpoints
Admin-only endpoints: products, orders, customers, coupons, offers, metrics, activities, activity-log, uploads. Apply admin rate limiter.

### Phase C — Frontend Shell
`/admin/layout.tsx` auth gate, `AdminSidebar`, token additions to globals.css, `admin-api.ts`.

### Phase D — Dashboard Tab (P1)
KPI cards with sparklines, Top Products, Recent Orders, Activity Feed.

### Phase E — Products Tab (P1)
Paginated table, Add/Edit modals with 3-image gallery, delete.

### Phase F — Orders Tab (P1)
Status filter tabs, table, detail sidebar with fulfillment transitions + tracking.

### Phase G — Customers, Offers, Reviews, Interface Billboard (P2)
All four P2 tabs wired to their respective API endpoints.

### Phase H — Analytics & Settings (P3)
Analytics chart, Profile update, Activity Log read-only table, Reports placeholder.

### Phase I — Polish & Validation
Audit events on all mutations, lint, build, quickstart walkthrough, security validation.
